Interfacepublic interface IEntityRepository

An EntityRepository is a collection of entities of a specific type and it can be thought of as a kind of local copy of the database. There is one EntityRepository for each entity in your application.


Example
To get a reference to a particular EntityRepository you need to use EntityManager.getRepository: 
  var userEntityRepository:IEntityRepository = em.getRepository(User);

Property detail
entitiesproperty
entities:EntityCollection  [read-only]

The collection of currently loaded and persisted entities. This is fully bindable as per normal Flex binding and can be assigned directly as a dataprovider to Flex components. The entities collection always exists, so its possible to bind a component to entities and then later load entities into a repository; once the entities have loaded the component will automatically update its view.

Implementation
    public function get entities():EntityCollection

Example
Binding to a List component 
   <s:List dataProvider="{em.getRepository(User).entities}" labelField="name" />

getEntityState()method 
public function getEntityState(entity:Object):String

Get the state of an entity. An entity which is not yet in a repository is NEW, an object which is in a repository is MANAGED, and object which is an identified entity but isn't in a repository is DETACHED and an object which is scheduled for removal is REMOVED.
