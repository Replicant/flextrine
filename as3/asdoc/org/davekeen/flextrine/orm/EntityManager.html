callRemoteMethod()method 
public function callRemoteMethod(methodName:String, ... args):AsyncToken

Call a custom method on a remote service. By default this will call a remote method on the default FlextrineService, but its also possible to specify a different service in the services folder by using the notation <remoteServiceName>.<remoteMethodName>

This method returns an AsyncToken to which you can attach responders in order to handle the result.

Parameters

methodName:String — The name of the method. This can either be in the form <remoteMethodName> or <remoteServiceName>.<remoteMethodName>
 
... args — A variable length list of arguments that will be passed directly to the remote method.

Returns
AsyncToken — An AsyncToken which can be used to handle a result or fault.

Example
Call a doSomething remote method in FlextrineService 
		 em.callRemoteMethod("doSomething");

Example
Call a doSomethingElse remote method on a service called MyNewService 
		 em.callRemoteMethod("MyNewService.doSomethingElse");

Example
Call the addTwoNumbers method on FlextrineService and handle the result 
		 em.callRemoteMethod("addTwoNumbers", 2, 3).addResponder(new AsyncResponder(onAddTwoNumbersResult, null));
		 
		 function onAddTwoNumbersResult(e:ResultEvent, token:Object):void {
		   trace("The answer is " + e.result);
		 }

detachCopy()method 
public function detachCopy(entity:Object):Object

Return an unmanaged copy of the given entity. This entity is not the same instance as that in the repository and changes to its properties will not trigger updates on the database. This has no effect on the entity given as a parameter - it remains in its repository as a fully managed entity. Detached copies still support on-demand loading and bidirectional association management.

Typically detachCopy would be used in situations where you want to make changes to an entity that might be discarded; for example, an edit window with a Save and Cancel button. If the user hits cancel the unmanaged entity can just be thrown away, or otherwise EntityManager.merge can be used to merge changes back into the repository.

Parameters

entity:Object — The entity to detach.

Returns
Object —

Example
To detach a user entity 
		 var detachedUserCopy:User = em.detachCopy(user) as User;

merge()method 
public function merge(entity:Object):Object

Merge a detached entity back into Flextrine. If the entity has changed compared to its merged counterpart the merged object will be marked dirty and scheduled for update on the next flush.

merge is the opposite operation to detach

Parameters

entity:Object — The entity to merge

Returns
Object — The managed entity. This will be a different instance to the entity that was passed into the method.

Example
Thie following example detaches an entity, makes a change, then merges it back into the repository 
		 var unmanagedUser:User = em.detach(managedUser) as User;
		 unmanagedUser.name = "A different name";
		 em.merge(unmanagedUser);
		 
		 // This will write the changes that were made to the entity whilst it was detached
		 em.flush();

requireMany()method 
public function requireMany(parentEntity:Object, manyAttributeNames:*, onResult:Function = null, onFault:Function = null, fetchMode:String = null):AsyncToken

Ensure that a many valued collection is loaded before taking an action. If the collection is already loaded this calls onResult instantly.

Parameters

parentEntity:Object — The entity containing the unitialized collection.
 
manyAttributeNames:* — The name of the collection attribute as a String, or an array of String attributes
 
onResult:Function (default = null) — A callback to invoke when the collection association has loaded.
 
onFault:Function (default = null) — A callback if the require fails.
 
fetchMode:String (default = null)

Returns
AsyncToken —

Example
To initalize a lazily loaded collection association toes on a foot entity 
		 em.requireMany(foot, "toes", onToesLoaded);
		 
		 function onToesLoaded():void {
		   trace("We loaded " + foot.toes.length + " toes");
		 }
It is also possible to use an array of attribute names if you need to required more than one association at a time. A common usage of this would be to wrap a block that works on possibly uninitialized collections in a requireMany call with all the associations: 
		 em.requireMany(foot, [ "toes", "nails" ], function():void {
		 	foot.toes.add(new Toe());
		  foor.nails.add(new Nail());
		 } );
